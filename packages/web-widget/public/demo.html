Support Widget — локальная демо-страница

Эта страница подключает собранный бандл widget.js и инициализирует SupportWidget для ручной проверки виджета.

Локальная разработка

  1. Задайте два ephemeral master key по runbook и поднимите локальный стек: docker compose up -d (API доступен через proxy на :8080).
  2. Соберите виджет: npm run widget:build из корня репозитория.
  3. Запустите preview с прокси: npm run preview --prefix packages/web-widget
  4. Откройте http://127.0.0.1:5174/demo.html и подставьте stage public key из npm run bootstrap:local.

apiBase

При preview через Vite proxy достаточно относительного пути /api/v1. Если открываете HTML напрямую без proxy, укажите полный URL: 

apiBase: "http://127.0.0.1:8080/api/v1"

Альтернатива — прямой доступ к API Gateway: http://127.0.0.1:4101/api/v1.

Embed snippet

<script src="https://cdn.example.com/widget.js"></script>
<script>
  SupportWidget.init({
    apiBase: "/api/v1",
    environment: "stage",
    publicKey: "sk_stage_...",
    externalId: "visitor-1"

</script>
